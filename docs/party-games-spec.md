# 聚会游戏 — 详尽设计规格

> 撰写日期：2026-07-09
> 状态：分析 & 规划

---

## 1. 概述

聚会游戏模块包含两个独立游戏：
- **谁是卧底**（Undercover）：社交推理，类似狼人杀/谁是卧底
- **真心话大冒险**（Truth or Dare）：轮流抽卡完成任务

两种模式共享房间系统（创建/加入/等待），但游戏逻辑完全不同。

## 3. Bug 清单（按严重程度排序）

### 🔴 B1 — 本地模式：真心话大冒险走入谁是卧底流程

**位置**：`useLocalGame.ts` → `startLocalGame()`

**现象**：选择"真心话大冒险"模式，创建房间后开始游戏，进入的是谁是卧底的词语→发言→投票流程。

**根因**：`startLocalGame` 没有检查 `room.settings.mode`，始终执行卧底逻辑。

**后端对比**：`partyRoomState.js` 的 `startPartyGame` 正确判断了模式（第 119-125 行），真相大冒险直接进入 `punishment` 阶段。

### 🔴 B2 — 真心话大冒险人数限制错误

**位置**：`useLocalGame.ts` → `startLocalGame()`

```
if (playerCount < 3) return // 最少 3 人
```

真心话大冒险 2 人即可，但这里硬编码了 3 人限制。

### 🟡 B3 — 真心话大冒险缺少独立游戏视图

真心话大冒险只用 `TruthOrDarePanel` 在惩罚阶段显示，没有独立的游戏主界面。应有一个完整的真心话大冒险视图，包括：
- 当前轮到谁
- 所有玩家的回合顺序
- 历史记录（谁做了什么任务）
- 更丰富的卡片选择交互

### 🟡 B4 — PhaseStepper 仅适配谁是卧底

`PhaseStepper.tsx` 硬编码了谁是卧底的五个阶段（词语→发言→投票→结果→惩罚）。真心话大冒险应该有不同的阶段显示。

### 🟡 B5 — UndercoverRoundView 名称误导

组件名为 `UndercoverRoundView` 但实际上在 `index.tsx` 中用于所有 `phase !== 'punishment'` 的游戏内容，包括真心话大冒险可能进入的意外流程。

### 🟡 B6 — 在线模式无描述同步

`DescriptionBoard` 的描述输入仅存储在客户端本地状态，未通过 WebSocket 同步给其他玩家。多人在线时看不到彼此的发言描述。

### 🟢 B7 — 缺少游戏历史/日志

没有记录每轮的结果、谁被淘汰、谁做了什么任务，无法回顾游戏过程。

### 🟢 B8 — 缺少音效/震动反馈

移动端游戏常见的触觉反馈和音效完全没有。

## 4. 游戏流程设计

### 4.1 谁是卧底 — 完整流程

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  等待室   │ → │ 查看词语  │ → │ 轮流发言  │ → │ 投票指认  │ → │ 结果揭晓  │ → │ 惩罚环节  │
│ waiting  │    │  word    │    │ speaking │    │  voting  │    │  result  │    │punishment│
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
                                                                                      │
                                                                                      ↓
                                                                              ┌──────────┐
                                                                              │  等待室   │ (新一轮)
                                                                              │ waiting  │
                                                                              └──────────┘
```

**各阶段详述**：

| 阶段 | 参与者 | 操作 | UI 要素 |
|------|--------|------|---------|
| **等待室** | 所有人 | 房主开始游戏 | 玩家座位、房间码、人数 |
| **查看词语** | 每人独立 | 点击查看/隐藏自己的词语 | 词语卡片（可隐藏）、身份标签 |
| **轮流发言** | 按顺序 | 每人描述自己的词（不可直说） | 当前发言人高亮、发言进度条、描述输入框、描述记录板 |
| **投票指认** | 所有人 | 投票给怀疑的卧底 | 可点击的玩家头像、已投票标记 |
| **结果揭晓** | 所有人 | 查看投票结果 | 淘汰者显示、胜负判定、描述回顾 |
| **惩罚环节** | 被淘汰者 | 抽取真心话/大冒险卡 | 卡片翻转、完成/换题按钮 |

**规则约束**：
- 最少 3 人，建议 4-8 人
- 1 名卧底，其余平民
- 卧底和平民的词语相似但不同
- 所有人发言完毕后投票
- 得票最多者出局
- 投出卧底 → 平民胜；卧底存活到最后 2 人 → 卧底胜

### 4.2 真心话大冒险 — 完整流程

```
┌──────────┐    ┌──────────────────────────────────────┐
│  等待室   │ → │          真心话大冒险循环              │
│ waiting  │    │                                      │
└──────────┘    │  ┌────────┐    ┌────────┐          │
                │  │ 选择类型 │ → │ 展示卡片 │ → 完成   │
                │  │ 真心话  │    │ 执行任务 │          │
                │  │ 大冒险  │    └────────┘          │
                │  │ 随机   │         │                │
                │  └────────┘         ↓                │
                │               ┌──────────┐          │
                │               │ 下一玩家  │ ←──────┘
                │               └──────────┘
                └──────────────────────────────────────┘
```

**流程详述**：

| 步骤 | 参与者 | 操作 | UI 要素 |
|------|--------|------|---------|
| **等待室** | 所有人 | 房主开始游戏 | 玩家座位、房间码 |
| **选择类型** | 当前轮到的玩家 | 选择真心话/大冒险/随机 | 三个大按钮、当前玩家高亮 |
| **展示卡片** | 当前玩家 | 查看卡片内容并执行 | 卡片展示（带翻转动画）、强度标签 |
| **完成/换题** | 当前玩家 | 确认完成或换一张 | 完成（下一人）/换题按钮 |
| **循环** | 所有人轮换 | 自动切换到下一玩家 | 玩家顺序指示器、已完成的标记 |

**规则约束**：
- 最少 2 人
- 无卧底/词语概念
- 纯粹轮流抽卡
- 完成一轮后自动继续下一轮
- 三种强度：轻松🌸 / 有趣😄 / 刺激🔥
- 退出房间即结束

## 5. 在线模式 vs 本地模式 功能矩阵

| 功能 | 在线模式 | 本地模式 |
|------|----------|----------|
| 创建房间 | ✅ API | ✅ 本地生成 |
| 加入房间 | ✅ 房间码 | ✅ 同一会话 |
| WebSocket 实时同步 | ✅ | ❌ 不需要 |
| 多设备多人 | ✅ | ❌ 单设备传手机 |
| 随机添加玩家 | N/A | ✅ |
| 玩家切换 | N/A | ✅ |
| 描述输入 | ⚠️ 本地存储 | ✅ |
| 描述同步（给其他玩家看） | ❌ 未实现 | ✅ 天然可见 |
| 游戏状态持久化 | ✅ Durable Object | ❌ 刷新即失 |

```
│   │
│   ├── CreateRoomSheet.tsx      # 创建房间

│   ├── WaitingRoomView.tsx      # 等待室

│   ├── TruthOrDareGameView.tsx  # ★NEW★ 真心话大冒险独立视图
│   ├── TruthOrDarePanel.tsx     # 惩罚面板（抽卡→展示→完成）
│   │
│   └── LocalModeBar.tsx         # 本地模式操作栏（从 index.tsx 抽出）
```

### 7.2 状态机修复 — `useLocalGame.ts`

`startLocalGame` 需要分叉：

```
if (mode === 'truth-or-dare') {
  → phase = 'punishment'
  → punishmentTargetId = players[0].id
  → truthOrDareTurnIndex = 0
} else {
  → assign undercover roles & words
  → phase = 'word'
}
```

### 7.3 渲染分支 — `index.tsx`

```
if (room.phase === 'waiting') → WaitingRoomView
else if (room.settings.mode === 'truth-or-dare') → TruthOrDareGameView
else → UndercoverGameView
```
